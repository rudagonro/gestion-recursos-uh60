Problema central

La gestión de recursos del UH-60 opera de forma reactiva porque no existe integración suficiente entre la planeación financiera, la logística y la gestión del ciclo de vida.

Idea fuerza para exposición

El problema principal no es solo presupuestal; es una brecha de arquitectura institucional que impide convertir datos, necesidades y recursos en una estrategia de sostenimiento de largo plazo.

Gobernanza
Dispersa

Planeación financiera
Anual

Uso de indicadores
Limitado
Las métricas no se usan de forma sistemática para ajustar recursos.
Modelo objetivo
Integrado
Programa UH-60 con gestión de ciclo de vida, CCV e indicadores.

3. Diagnóstico institucional

Comparación entre el modelo actual y el modelo objetivo para la gestión del ciclo de vida.

Actual vs ideal

Modelo actual · Gestión reactiva

  • Requerimientos operativos → logística → presupuesto anual.
  • Énfasis en reposición, contratación y solución de urgencias.
  • Seguimiento de ejecución, pero poco uso del impacto para decidir recursos.
  • El sistema responde a la demanda más que a una hoja de ruta de ciclo de vida.
→

Modelo ideal · Gestión integrada

  • Programa UH-60 como nodo articulador de sostenimiento.
  • Planeación plurianual alineada con mantenimiento, suministro y modernización.
  • Indicadores, costos y riesgos financieros como base de priorización.
  • Gestión del ciclo de vida para mejorar disponibilidad, eficiencia y costo-efectividad.

4. Recomendaciones estratégicas

Cuatro líneas para migrar del diagnóstico a un modelo de sostenimiento maduro.

Plan de acción

Crear Oficina del Programa UH-60

  • Centraliza gobernanza, decisiones y seguimiento.
  • Conecta mantenimiento, finanzas, logística e ingeniería.

Implementar planeación plurianual

  • Vincula contratos, TBO, fases mayores y stocks críticos.
  • Reduce reprogramaciones y volatilidad del alistamiento.

Usar indicadores de desempeño

  • Disponibilidad, costo por hora, tiempos de reparación y stock crítico.
  • Convierte datos en decisiones de asignación de recursos.

Operacionalizar el CCV

  • Proyecta costos de mantenimiento, modernización y obsolescencia.
  • Permite priorizar con visión de ciclo de vida.

Interactividad
Este dashboard resume el diagnóstico del Apéndice L: muestra que la gestión de recursos del UH-60 funciona hoy de manera reactiva porque la planeación financiera, la logística y la gestión del ciclo de vida no operan como un solo sistema institucional integrado.
La causa raíz es una brecha de integración: hay requerimientos, ejecución y seguimiento, pero no existe un nodo central del programa que convierta esos insumos en planeación plurianual, criterios de priorización y decisiones soportadas por indicadores y costo de ciclo de vida.
El modelo ideal cambia la lógica del sistema: en lugar de empezar por necesidades logísticas y luego buscar presupuesto, el Programa UH-60 planifica el ciclo de vida, define prioridades, proyecta costos y alinea los recursos con la disponibilidad operacional esperada.
